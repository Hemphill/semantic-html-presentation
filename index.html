What is Semantic HTML?

Semantic HTML is the judicial application of HTML tags to identify individual pieces of content, and to express organization of information on a webpage.

Rather than simply defining the presention or styling, semantic HTML provides context, meaning, and organizational structure to the content on a webpage.

key Benefits

  • Cleaner Code:
    easier to edit & maintain. Saves time, effort, and money!
  • SEO:
    search engines can crawl the webpage and make sense of it.
  • Accessibility:
    assistive technologies (e.g. screen readers) can more easily navigate and present the content.

WCAG 2.0

Guideline 1.3

“Create content that can be presented in different ways (for example simpler layout) without losing information or structure.”

Characteristics of
Accessible HTML

  • semantic treatment of content
  • proper use of attributes and textual alternates
  • sensitivity to contrast, colors, and font sizes

Well Crafted HTML is
Semantic HTML

HTML markup is meant to define the structure of the document and add context to the content.

Two Basic parts of
semantic HTML

  1. Content Markup
  2. Document Structure

Content Markup

Using the appropriate HTML tag for the content.
Giving content context and meaning.

Is it a paragraph of text? Use a <p> tag.
Is it a heading? Use a <h1> to <h6> tag.
Is it a list of items? Use <li> tags.

Document Structure

Organizing and providing structure to the document.
Aids in organization and navigation of content.

Is the section a page header? Use a <header> tag.
Is the section the main content area? Use a <main> tag.
Is the section a navigation bar? Use <nav> tags.

Accessible HTML Tips

These are some simple tips that will help accessibility.

  1. Use the alt="" attribute on images
  2. Use <label> tags to title form elements

Good Semantic Example


<header>
  <h4>Greg's Awesome Web Guide</h4>
  <nav>
    <ul>
      <li><a href="/home">Home</a></li>
      <li><a href="/contact">Contact</a></li>
    </ul>
  </nav>
</header>
<main>
  <article>
    <h1>Semantic HTML</h1>
    <blockquote>
      Semantic HTML is the use of HTML markup to reinforce the semantics, or meaning, of the information in webpages and web applications rather than merely to define its presentation or look. Semantic HTML is processed by traditional web browsers as well as by many other user agents.
      <cite>
        <a href="https://en.wikipedia.org/wiki/Semantic_HTML">
          Wikipedia on Semantic HTML
        </a>
      </cite>
    </blockquote>
    <p>
      Well crafted HTML leads easier to style, maintain, and more accessible web pages. HTML is meant to define the structure of the document and add context to the content.
    </p>
    <h2>Well Crafted HTML</h2>
    <ul>
      <li>semantic treatment of content</li>
      <li>proper use of attributes and textual alternates</li>
      <li>ensitivity to contrast, colors, and font sizes</li>
    </ul>
  </article>
</main>
<footer>
  <nav>
    <ul>
      <li><a href="/home">Home</a></li>
      <li><a href="/contact">Contact</a></li>
    </ul>
  </nav>
</footer>
